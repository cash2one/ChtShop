茶汇通APP接口文档

规则

1.请求数据全部为JSON格式输出

2.获取方式主要为POST和GET（若为GET方式会在URL处标记）

3.POST方式提交Version字段将会获取到不同版本的接口格式

4.不同版本返回的数据结构不一样。 * v1.

请求成功

  {
    code: 200 响应码，固定为200
    datas:  数据实体
  }

请求失败

  {
    code: 200
    datas: {
      error: "失败描述"
    }
  }

  • v2.

    请求成功

    {
  code: 200 响应码，200时表示请求数据成功，404表示请求失败
  content: 数据实体
}

    请求失败

    {
    code: 404 请求失败响应码
    content: 错误描述
}

  • v3.

    请求成功

    {
  code: 1 请求成功响应码
  msg:  结果描述
  data: 数据实体
}

    请求失败

    {
  code: 0 请求失败响应码 
  msg: 错误详情描述
}

{
  code: -1 用户未登录响应码
  msg: "请登录"
}

URL规则

BASE_URL:http://api.chahuitong.com

例如登录接口为:http://api.chahuitong.com?act=login

发送注册短信验证码

URL: act=login&op=send_code

参数：

mobile 手机号
type 0-注册 1-忘记密码

返回：

  {
      "code": 1,
      "msg": "验证码已发出，请注意查收",
      "data": true
  }

注册

URL: act=login&op=register

参数：

mobile 手机号
password 密码
code 验证码
client 客户端类型(android wap wechat ios)

返回：

  {
      "code": 1,
      "msg": "SUCCESS",
      "data": {
          "username": "12345678912",
          "key": "bb6a573583605063443c65f4395e559a"
      }
  }

登录

URL：act=login

参数：

username 用户名
password 密码
client 客户端类型(android wap wechat ios)

返回

  {
    code: 200
    datas: {
      username 用户名
      key 登录令牌
    }
  }

注销登录

URL：act=logout

参数：

mobile 手机号
client 客户端类型(android wap ios wechat)

返回

  {
    code: 1
    data: '1'
    msg:'SUCCESS'
  }

第三方登录

URL: act=login&op=oauth

参数：

client 客户端类型，可选：android，ios，web
type 第三方类型，可选：sina,wechat,qq
openid 第三方授权返回的唯一标识,如微博登录的Uid，腾讯的openId
username 用户在第三方平台的用户名或者昵称
